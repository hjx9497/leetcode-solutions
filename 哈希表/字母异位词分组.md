字母异位词分组 (LeetCode 49)
题目描述
给定一个字符串数组 strs，将 字母异位词 组合在一起。字母异位词指字母相同，但排列不同的字符串。可以按任意顺序返回结果列表。

示例：

text
输入: strs = ["eat", "tea", "tan", "ate", "nat", "bat"]
输出: [["bat"],["nat","tan"],["ate","eat","tea"]]
说明：

所有输入均为小写字母。

不考虑答案输出的顺序。

解题思路
核心思想
字母异位词的特点：排序后的字符串相同，或者 字符出现次数的统计结果相同。
因此可以使用哈希表，将 “标准化后的键” 映射到 原字符串列表。

方法一：排序法（简洁）
对每个字符串 s，将其字符排序得到 key。

使用 Map<String, List<String>>，将 key 相同的字符串放入同一个列表。

最后返回 map.values() 的集合。

方法二：计数法（更高效）
对每个字符串 s，统计每个字母的出现次数（26个字母），将计数序列（如 "1#2#0#..."）作为键。

使用相同的哈希表映射。

两种方法的时间复杂度都是 O(n × k log k) 或 O(n × k)，空间复杂度 O(n × k)。

复杂度分析
方法	时间复杂度	空间复杂度
排序法	O(n × k log k)	O(n × k)
计数法	O(n × k)	O(n × k)
n 是字符串数组长度，k 是字符串的最大长度。

计数法理论上更优，但实际中排序法通常足够快且代码简单。

代码实现
排序法（推荐，简洁易懂）
```java
import java.util.*;

public class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> map = new HashMap<>();
        
        for (String s : strs) {
            // 将字符串转为字符数组并排序
            char[] chars = s.toCharArray();
            Arrays.sort(chars);
            String key = new String(chars);
            
            // 放入哈希表
            map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }
        
        return new ArrayList<>(map.values());
    }
}
```
计数法（适合长字符串，避免排序）
```java
public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    
    for (String s : strs) {
        int[] count = new int[26];
        for (char c : s.toCharArray()) {
            count[c - 'a']++;
        }
        // 构造键，例如 "1#2#0#0#..."
        StringBuilder sb = new StringBuilder();
        for (int i = 0; i < 26; i++) {
            sb.append(count[i]).append('#');
        }
        String key = sb.toString();
        
        map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
    }
    
    return new ArrayList<>(map.values());
}
```
关键点 / 易错点
键的唯一性：排序后的字符串或计数序列必须能唯一标识一组字母异位词。

computeIfAbsent：简化了“若键不存在则创建新列表”的操作。

返回类型：new ArrayList<>(map.values()) 将 Collection<List<String>> 转换为 List<List<String>>。

小写字母限制：如果字符集扩大（如包含大写或 Unicode），计数数组大小需调整。
