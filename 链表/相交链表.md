# 相交链表 (LeetCode 160)

## 题目描述
给你两个单链表的头节点 `headA` 和 `headB`，请你找出并返回两个单链表相交的起始节点。如果两个链表没有交点，返回 `null`。

题目保证整个链式结构中不存在环。函数返回结果后，链表必须保持其原始结构。

**示例：**
```
链表 A: [4,1,8,4,5]
链表 B: [5,0,1,8,4,5]
相交节点: 8（值为 8 的节点）
```
**注意：** 交点不是数值相等，而是节点引用（内存地址）相同。

## 解题思路（哈希表法）

### 核心思想
使用哈希表（`HashSet`）记录链表 A 的所有节点。然后遍历链表 B，检查当前节点是否在哈希表中：
- 如果在，说明该节点是交点，直接返回。
- 如果遍历完 B 都没有找到，返回 `null`。

### 手动模拟
链表 A：`4 → 1 → 8 → 4 → 5 → null`  
链表 B：`5 → 0 → 1 → 8 → 4 → 5 → null`

1. 遍历 A，将节点 `4,1,8,4,5` 存入哈希表。
2. 遍历 B：
   - `5` 不在表中 → 继续
   - `0` 不在表中 → 继续
   - `1` 不在表中？等等，`1` 在 A 中（第二个节点），所以 `1` 是交点吗？  
     **注意：** 这里的 `1` 节点在 A 和 B 中虽然值相同，但实际上是不同的节点对象（内存地址不同），因此不会被认为是交点。真正相交的是节点 `8`，因为 A 的 `8` 和 B 的 `8` 是同一个节点（后续的 `4` 和 `5` 也共享）。
   - 继续走到 `8`，发现 `8` 在表中 → 返回 `8`。

## 复杂度分析
- **时间复杂度：O(m + n)**，需要遍历两个链表各一次。
- **空间复杂度：O(m)** 或 **O(n)**，取决于选择存储哪个链表的节点。

## 代码实现（哈希表法）

```java
import java.util.HashSet;
import java.util.Set;

public class Solution {
    public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        Set<ListNode> visited = new HashSet<>();
        ListNode curr = headA;
        while (curr != null) {
            visited.add(curr);
            curr = curr.next;
        }
        curr = headB;
        while (curr != null) {
            if (visited.contains(curr)) {
                return curr;
            }
            curr = curr.next;
        }
        return null;
    }
}
```

## 关键点 / 易错点
- **比较的是节点引用，不是节点值**：两个不同节点即使值相同也不相交。
- **空链表处理**：如果任一链表为空，直接返回 `null`。
- **哈希表法空间 O(n)**，可以进一步优化为 O(1) 空间的双指针法。

## 扩展：双指针法（空间 O(1)）

### 核心思想
- 指针 `pA` 遍历链表 A，`pB` 遍历链表 B。
- 当 `pA` 到达末尾时，将其重定向到 `headB`；当 `pB` 到达末尾时，将其重定向到 `headA`。
- 如果两链表相交，`pA` 和 `pB` 会在相交节点相遇（因为各自走过的总长度都是 `lenA + lenB`，且最后一段公共部分长度相同）。
- 如果不相交，最终会同时为 `null` 并退出循环。

### 代码实现

```java
public class Solution {
    public ListNode getIntersectionNode(ListNode headA, ListNode headB) {
        if (headA == null || headB == null) return null;
        
        ListNode pA = headA;
        ListNode pB = headB;
        
        while (pA != pB) {
            pA = (pA == null) ? headB : pA.next;
            pB = (pB == null) ? headA : pB.next;
        }
        return pA;
    }
}
```

### 手动模拟（双指针法）
链表 A：`4 → 1 → 8 → 4 → 5 → null`（长度 5）  
链表 B：`5 → 0 → 1 → 8 → 4 → 5 → null`（长度 6）  
公共部分从 `8` 开始，长度 3。

| 步骤 | pA        | pB        |
|------|-----------|-----------|
| 初始 | 4         | 5         |
| 1    | 1         | 0         |
| 2    | 8         | 1         |
| 3    | 4         | 8（相遇？不，此时 pB=8，pA=4，值不同但对象不同） |
| 4    | 5         | 4         |
| 5    | null      | 5         |
| 6    | headB(5)  | null      |
| 7    | 0         | headA(4)  |
| 8    | 1         | 1         |
| 9    | 8         | 8 → 相遇，返回 8 |

## 两种方法对比

| 维度 | 哈希表法 | 双指针法 |
|------|----------|----------|
| 空间复杂度 | O(m) 或 O(n) | O(1) |
| 代码长度 | 稍长 | 极短 |
| 理解难度 | 简单直观 | 需要数学证明 |
| 推荐场景 | 面试时先说思路 | 最终提交的最优解 |
